Autonomous Systems Theory

What are Autonomous Systems?

An Autonomous System (AS) is a collection of connected Internet Protocol (IP) routing prefixes under the control of one or more network operators on behalf of a single administrative entity or domain.

Key Characteristics:

  • Unique AS Number (ASN): Each AS has a unique identifier (e.g., AS1, AS2)
  • Single Administrative Domain: Managed by one organization
  • Common Routing Policy: Uses consistent routing decisions
  • Interior Gateway Protocol: Runs protocols like OSPF or RIP internally

Types of Autonomous Systems:

  • Stub AS: Connected to only one other AS (like small organizations)
  • Transit AS: Carries traffic between other ASes (like major ISPs)
  • Multi-homed AS: Connected to multiple ASes but doesn't carry transit traffic

Routing Between ASes:

ASes communicate using the Border Gateway Protocol (BGP), which determines the best paths for data to travel across the internet. When nodes fail, BGP automatically finds alternative routes.

In This Simulation:

You're managing packet routing through a network of ASes. Red nodes represent failed systems, and you must find valid paths from Sender (S) to Receiver (R) using available connections.
